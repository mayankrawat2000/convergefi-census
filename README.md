# Census Document Q&A Chatbot

This is an agentic Document Q&A Chatbot built to query, summarize, and generate structured artifacts (tables, charts) from 2011 Census Highlight documents for Karnataka, Odisha, and Madhya Pradesh.

## Tech Stack
- **Backend**: Python 3.11, FastAPI, Uvicorn, SQLite
- **Frontend**: Streamlit
- **LLM Engine**: DeepInfra OpenAI-compatible API (Llama-3.3-70B-Instruct or Qwen-2.5-Coder-32B-Instruct)
- **Deployment**: Docker Compose

---

## Quick Start (Docker Compose)

The easiest way to run the entire system is using Docker Compose. It will build and launch both the FastAPI backend and Streamlit frontend.

### 1. Configure Environment
Create a `.env` file in the root directory (one is already provided) and set your DeepInfra API key:
```env
DEEPINFRA_API_KEY="YOUR_DEEPINFRA_API_KEY"
```

### 2. Launch the Containers
Run the following command to build and launch the application:
```bash
docker compose up --build
```
Once the containers are running:
- **Streamlit Frontend**: Access at [http://localhost:8501](http://localhost:8501)
- **FastAPI Backend API**: Access at [http://localhost:8000](http://localhost:8000)

## Local Setup (Without Docker)

If you prefer to run the services locally in your virtual environment (`myenv`):

### 1. Install Dependencies
Ensure you have the virtual environment activated and run:
```bash
pip install -r requirements.txt
```

### 2. Parse Census Data
Run the data preparation script to parse the markdown reports, index pages in SQLite, and output structured tables:
```bash
python app/data_prep.py
```
This compiles the raw reports into:
- A SQLite database: `workspace/census_data.db`
- A flat master district CSV: `workspace/data/census_master.csv`
- Individual statement table CSVs: `workspace/data/<State>/statement_<number>.csv`

### 3. Run FastAPI Backend
Start the FastAPI server on port 8000:
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 4. Run Streamlit Frontend
Start the Streamlit interface on port 8501:
```bash
streamlit run streamlit_app/app.py
```

## Running Tests

We have written comprehensive unit tests covering:
1. **Python Executor Sandbox**: Validates running python scripts, capturing outputs, errors, timeouts, and Matplotlib chart generation.
2. **Database Search & Page Retrieval**: Tests page indexing, keyword search with logical AND, and page retrieval.
3. **Agent Conversational engine**: Verifies citation parsing and response structuring.

To run the tests, execute:
```bash
python -m pytest tests/
```

## How It Works & Key Features

1. **Agent Reasoning Loop**: The agent utilizes tool-calling to query census documents. When asked quantitative or comparison questions, the agent lists available CSV tables and writes Python scripts using `pandas` to get mathematically precise answers.
2. **Execution Logs**: You can view what the agent did behind the scenes by expanding the "Agent Thinking Trace" dropdown in the Streamlit UI to inspect subprocess calls, python stdout/stderr, and search queries.
3. **Inline Artifacts**: Charts (Matplotlib plots) and tabular markdown generated by the agent are displayed directly in the chat message bubble.
4. **Conversation Memory**: The chatbot stores history locally in `workspace/history/` under unique session IDs, allowing you to ask follow-up questions referencing earlier turns.
5. **Citations**: Claims are backed by source citations, identifying the document, page number, and original text snippet.
